# 标题：Files.md：一个用纯Markdown文件管理生活的开源应用

## 📋 基本信息
- **来源资讯**: GitHub / Hacker News
- **阶段**: 采集

## 💡 核心观点
一款名为Files.md的开源应用，可以让用户在纯Markdown文件中管理整个数字生活——笔记、文档、日记、任务，全部离线存储，AI友好。

## 📖 背景信息
Files.md是由开发者 zakirullin 创建的一个轻量级应用，用户可以在浏览器中管理和编辑本地Markdown文件。它不向服务器发送任何数据，所有内容存储在用户设备上，支持离线使用，可以与iCloud、Dropbox、Google Drive同步，或者自托管部署。这个项目的核心理念是：你的数据应该属于你自己，软件应该服务于数据而不是相反。目前它在GitHub上获得了超过500颗星，并得到了HN社区的高度关注。

## 📄 文章
在AI助手越来越聪明的今天，你的个人数据在哪里？这个问题很少有人认真思考。

开发者 zakirullin 用他创建的项目 Files.md 给出了自己的回答。他的理念很简单：把你的生活整理成一个个 Markdown 文件，让AI围绕你的数据工作，而��是反过来。

Files.md 是一个极其简洁的Web应用。用户不需要安装任何东西，只需在浏览器中打开 app.files.md，选择一个本地文件夹，然后就可以开始添加笔记、文档、日记、任务列表。所有数据都以标准的Markdown格式存储在用户的设备上，不会有任何内容发送到服务器。

这种"本地优先"的设计有几个重要优势。首先是隐私——你的数据永远在你手中，不会被任何云服务收集。其次是持久性——Markdown是一种开放格式，即使几十年后，只要有文本编辑器，你仍然可以读取这些文件。第三是AI友好——LLM本身就擅长理解和生成Markdown格式的文本，你的笔记可以直接喂给AI使用。

与Obsidian、Notion等流行的笔记应用相比，Files.md刻意放弃了复杂的功能。它没有双向链接图谱，没有插件系统，没有花哨的主题。作者认为，限制反而能促进创造力。就像写文章不需要Word的全部功能一样，管理日常生活也不需要那么复杂的工具。

开发者分享了他使用这套系统的亲身体验：他将笔记分为"大脑"和"开发"两个文件夹，在使用中发现某些跨领域的知识点产生了有趣的连接，最终促成了他的文章《软件开发中的认知负荷》。这种深度思考的过程，比使用任何高级笔记工具都更重要。

这个项目获得500多颗星，排在HN当日热门榜前列。社区的反应说明，很多人跟他一样，正在寻找一种更简单、更自主的数字生活方式。在AI日益侵入我们数字生活的时代，这种"数据主权"的诉求显得格外重要。

## 🔄 二阶效应
本地优先的应用开发潮流兴起；Markdown作为数据格式的地位进一步巩固；个人知识管理工具与AI的结合成为新趋势。
